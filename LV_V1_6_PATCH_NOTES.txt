## v1.6.1 R2 AutoSync emergency patch

이번 패치는 기존 R2 버킷 구조를 CMS가 자동으로 읽지 못해 기본/샘플 화면만 보이는 문제를 해결합니다.

### 핵심 변경
- `/api/backup` 실행 시 R2 `stores/` 폴더를 자동 스캔합니다.
- R2 구조 `stores/<store>/left/*`, `stores/_common/right/*`를 자동으로 D1 `stores`, `contents`, `devices`에 인덱싱합니다.
- `/api/playlist`는 D1에 데이터가 없으면 R2를 직접 읽어 fallback합니다.
- `/api/player-config`도 D1 데이터가 없으면 R2를 fallback으로 사용합니다.
- `/api/r2-sync` 신규 추가: 현재 R2 구조를 직접 확인하고 D1에 동기화합니다.
- `/api/health`에서 DB/MEDIA/R2_PUBLIC_BASE/ONLINE_TTL_SEC 바인딩 상태를 확인할 수 있습니다.

### Cloudflare Pages 필수 바인딩
- D1 binding name: `DB`
- R2 binding name: `MEDIA`
- R2 bucket: `localvision-media-prod` 또는 실제 운영 버킷
- Variables:
  - `R2_PUBLIC_BASE`: 공개 R2 base URL
  - `ONLINE_TTL_SEC`: `600`

### 배포 후 확인 URL
- `/api/health`
- `/api/r2-sync`
- `/api/backup`
- `/api/playlist?store=goobne&side=left`
- `/api/playlist?store=goobne&side=right`
